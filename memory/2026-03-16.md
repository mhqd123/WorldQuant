# 2026-03-16

- Reached a key workflow conclusion for WorldQuant BRAIN research: optimizing for raw simulation score alone is flawed. The correct objective must jointly consider signal strength, self-correlation, coupling to existing factor families, and real submission conversion.
- User proposed a better strategy than brute-force breadth: first identify a near-submittable “mother factor”, then iteratively optimize that single line until it is genuinely submit-capable; only after that should the process expand to different factor families. This was adopted as the new guiding research direction.
- Important conceptual distinction established:
  - self-correlation = platform-level similarity constraint against existing account alphas
  - coupling = research-level overlap in mechanism/family, even before the platform explicitly rejects it
  Future factor evaluation should treat both as first-class dimensions alongside Sharpe/Fitness.

- Confirmed by live API querying that the account contains a very large unsubmitted pool (~4106 UNSUBMITTED alphas), with many high-score candidates never yet attempted for formal submission. A comparison against local submit-attempt logs showed hundreds of candidates meeting the working visible-metric screen but not yet formally tried.
- Confirmed that many formal submission attempts return HTTP 201 yet remain `UNSUBMITTED` with `dateSubmitted = null` and `SELF_CORRELATION = PENDING`; this means the request hit the platform successfully but did not complete into a finalized accepted submission. Distinguish carefully between “submit request accepted” and “submission actually landed”.
- Also confirmed platform throttling behavior: batch formal submissions can hit HTTP 429 `THROTTLED`, while single-item submission often succeeds. Going forward, safer submission policy is one-at-a-time or very low-rate serial submission.
- Simulation concurrency limits also matter on the platform. User screenshot showed manual simulations can fail simply because the account already reached the concurrent simulation limit; this is separate from formula correctness.
- A promising mother-factor direction was found: short-horizon mean-reversion / volume-confirm family still has strong raw signal, but many variants are too crowded. One tested variant achieved roughly Sharpe 2.53 and Fitness 1.28 but failed formal submission due to self-correlation about 0.8337, showing the signal family is real but over-coupled.
- User manually tested three first-generation de-correlation variants around that mother factor. Lessons:
  - changing only the volatility penalty (e.g. returns volatility to high-low range volatility) did not reduce self-correlation enough
  - changing only the volume-confirm transform tended to damage Fitness
  - therefore the next optimization focus should be altering the core price signal definition itself, not just risk term or volume wrapper
- Recommended next exploration for the mother-factor path: keep the volume confirmation idea but replace the price backbone with materially different definitions such as relative deviation from moving mean, smoothed trend-delta, or intraday close-position terms.
- Important user correction to keep explicit in future work: the real objective is submission to the WorldQuant BRAIN platform, not git commits. Local git commits are only engineering checkpoints for preserving workflow/code changes and must never be treated as task completion or emphasized over platform submission progress.
